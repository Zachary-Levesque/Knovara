# Roadmap

## Overview

Knovara's long-term vision is to become the intelligence layer for organizational knowledge.

However, the product will be developed incrementally.

Each phase validates a specific hypothesis before expanding scope.

---

# Phase 1: Technical Knowledge Discovery

## Goal

Help engineers understand technical projects faster.

## Hypothesis

Engineers spend significant time searching for and understanding technical information before they can contribute effectively.

## Features

* Repository analysis
* Documentation ingestion
* Technical summaries
* Learning recommendations
* AI-powered Q&A

## Success Metrics

* Reduced time to understand a project
* Positive user feedback
* Repeated usage

---

# Phase 2: Organizational Context

## Goal

Help engineers understand the people and teams behind technical systems.

## Hypothesis

Understanding ownership and expertise is as important as understanding code.

## Features

* Team mapping
* Ownership discovery
* Expert identification
* Project relationships

## Success Metrics

* Faster expert discovery
* Reduced onboarding questions
* Increased user engagement

---

# Phase 3: Decision Intelligence

## Goal

Help engineers understand why systems were built the way they were.

## Hypothesis

Historical context is one of the hardest forms of knowledge to discover.

## Features

* Decision tracking
* Architecture rationale
* Historical project context
* Knowledge timelines

## Success Metrics

* Reduced repeated questions
* Improved understanding of systems
* Increased confidence among new engineers

---

# Phase 4: Personalized Technical Mentorship

## Goal

Provide every engineer with a personalized path to productivity.

## Hypothesis

Engineers need guidance, not just information.

## Features

* Personalized onboarding plans
* Skill gap identification
* Learning recommendations
* Progress tracking

## Success Metrics

* Faster ramp-up
* Increased productivity
* Improved onboarding experience

---

# Phase 5: Organizational Intelligence Platform

## Goal

Become the central interface for organizational knowledge.

## Features

* Unified organizational knowledge graph
* Cross-team discovery
* Organizational insights
* Workforce intelligence

## Success Metrics

* Organization-wide adoption
* Reduced knowledge silos
* Measurable productivity improvements

---

# Current Focus

The current focus is validating the core problem:

> Engineers spend too much time understanding an organization before they can contribute effectively.

The first product should be as simple as possible while still proving this hypothesis.

Everything else is secondary.
